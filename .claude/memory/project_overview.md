IMT-MP is a CSM (Cities Skylines Multiplayer, workshop ID `1558438291`) sync extension for the
Intersection Marking Tool mod (IMT, workshop ID `2140418403`). It broadcasts IMT user-actions
across all clients in a multiplayer session so road markings stay in sync. Without it, IMT
markings only exist on the local client and diverge during play.

**Why:** The user's Loading Screen Mod report showed ~111 failed-to-load assets and ~158 missing
dependencies. The single highest-impact missing dependency was workshop ID `2764081051` (a JP
road-marking pack), referenced by 78 of the failed assets. That pack is generated by IMT — but
IMT had no CSM sync extension, so even subscribing to it wouldn't help in multiplayer. This
project unblocks that.

**How to apply:** When making changes here, default to behavior consistent with the existing
sync-extension family (CSM.MoveItSync, CSM.TmpeSync, CSM.ExtraLandscapingTools). Follow CSM's
canonical `SampleExternalMod` layout: `Mod/`, `Patcher.cs`, `Injections/`, `Commands/`,
`Handlers/`, `Services/`.

**Repo layout:**
- Working tree: `M:\develop\IMT-MP\`
- Git directory (separate-git-dir): `M:\git\IMT-MP\`
- The `.git` file in the working tree is a pointer file containing `gitdir: M:/git/IMT-MP`
- Plan doc: `C:\Users\Brewing Storm Lite\.claude\plans\mutable-chasing-falcon.md`
- See [[build_and_deploy]] for build commands

**Upstream CSM source:** https://github.com/CitiesSkylinesMultiplayer/CSM —
`examples/SampleExternalMod/` is the canonical extension layout. Their wiki at
`/wiki/Modding-API` and `/wiki/Developer-Resources` cover the Connection/CommandHandler contract.
